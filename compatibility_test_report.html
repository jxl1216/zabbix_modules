3. 发现的问题及修复方案

问题 1: manifest_version 不匹配导致模块无法加载 (Zabbix 6.0)

现象: 模块页面显示 "Class not found."

根因: 本地 manifest.json 使用 manifest_version: 2.0 (适用于 Zabbix 6.4+/7.0)，但 Zabbix 6.0 的 CModuleManager::MAX_MANIFEST_VERSION = 1，2.0 > 1 导致 loadManifest() 返回 null，模块完全不会被加载。

影响版本: 仅 Zabbix 6.0

修复方案

部署到 Zabbix 6.0 时，将 manifest.json 中的 manifest_version 改为 1.0。部署脚本自动处理版本差异。

问题 2: CModule 命名空间差异 (Zabbix 6.0)

现象: Module.php "Class not found" 错误

根因: Zabbix 6.0 使用 Core\CModule 命名空间，而 Zabbix 6.4+/7.0 使用 Zabbix\Core\CModule。

影响版本: 仅 Zabbix 6.0

修复方案

Module.php 使用条件类定义:

if (class_exists('Zabbix\\Core\\CModule', false)) { class Module extends \Zabbix\Core\CModule { use ModuleInitTrait; } } else { class Module extends \Core\CModule { use ModuleInitTrait; } }
问题 3: SID 验证导致 "Access denied" (Zabbix 6.0)

现象: 模块页面显示 "Access denied. You are logged in as Admin. You have no permissions to access this page."

根因: Zabbix 6.0 的 CController 默认 $validate_sid = true (private 属性)。对于 GET 请求，checkSID() 检查 raw_input 中的 sid 参数，但 GET 请求的 raw_input 为 null，导致 checkSID() 返回 false，触发 access_deny()。

影响版本: 仅 Zabbix 6.0 (6.4+/7.0 不受影响)

修复方案

所有控制器的 init() 方法中添加 disableSIDvalidation() 调用 (使用 method_exists() 包装以确保兼容性):

public function init(): void { if (method_exists($this, 'disableCsrfValidation')) { $this->disableCsrfValidation(); } if (method_exists($this, 'disableSIDvalidation')) { $this->disableSIDvalidation(); } }
问题 4: Autoloader 目录大小写处理

现象: 调查过程中曾怀疑大小写问题

根因: Zabbix 6.0 autoloader 的 loadClass() 使用 strtolower(array_pop($chunks)) 将目录名转为小写。因此 actions/ (小写) 是正确的目录名。Zabbix 6.4+/7.0 的 autoloader 行为相同。

结论: 无需修改，保持 actions/ 和 views/ 小写即可。

5. 部署注意事项

部署到 Zabbix 6.0 时必须:
  • 将 manifest.json 中的 manifest_version 改为 1.0
  • 模块目录放在 /usr/share/zabbix/modules/HostBatchClone/
  • 通过 UI "Scan directory" 注册模块，或直接在 module 表中插入记录
  • 确保 modules.default_access = 1 (Super admin 角色默认已设置)
部署到 Zabbix 6.4+ / 7.0 时必须:
  • manifest.json 使用 manifest_version: 2.0 (本地默认值)
  • 模块目录放在 /usr/share/zabbix/ui/modules/HostBatchClone/
  • 通过 UI "Scan directory" 注册模块

6. 测试结论

HostBatchClone 模块 (v1.2) 在 Zabbix 6.0.47、6.4.x 和 7.0.x 三个版本上均通过全部功能测试。通过以下兼容性措施实现了跨版本支持:

  • CompatHelper.php — 运行时 API 差异处理
  • Module.php 条件类定义 — CModule 命名空间兼容
  • 控制器 init() 双重禁用 — disableCsrfValidation + disableSIDvalidation
  • 部署脚本 — 自动调整 manifest_version 和模块路径
